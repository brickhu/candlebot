# 路由重构说明

## 变更概述

已按照要求完成路由结构重构：

### 路由配置变更
```
原结构：
/          → Home页面（引导页）
/dashboard → Dashboard页面
/new       → 不存在

新结构：
/          → Dashboard页面（直接作为首页）
/dashboard → Dashboard页面（保持兼容）
/new       → 新建分析页面
```

### 页面结构调整
1. **移除单独的Home页面**：`/` 直接指向Dashboard
2. **新增NewAnalysis页面**：`/new` 专门用于新建分析
3. **Dashboard页面简化**：移除弹窗逻辑，只保留仪表板功能

## 文件变更

### 新增文件
1. `app/src/pages/NewAnalysis.jsx` - 新建分析页面
   - 包含完整的分析弹窗
   - 处理用户认证检查
   - 支持浏览器扩展跳转

### 修改文件
1. `app/src/index.jsx` - 路由配置更新
   - `/` 路由指向Dashboard
   - 新增 `/new` 路由指向NewAnalysis
   - 移除Home页面导入

2. `app/src/pages/Dashboard.jsx` - 简化Dashboard
   - 移除NewAnalysisModal导入
   - 移除弹窗状态管理
   - "新建分析"按钮改为跳转到`/new`
   - 扩展跳转处理改为跳转到`/new`

3. `app/src/components/NewAnalysisModal.jsx` - 保持不变
   - 仍然处理扩展数据预填充
   - 在NewAnalysis页面中使用

### 未使用文件（可考虑删除）
1. `app/src/pages/Home.jsx` - 原首页（不再使用）
2. `app/src/pages/HomeRedirect.jsx` - 创建的重定向组件（备用）

## 功能流程

### 新建分析流程
1. 用户在Dashboard点击"New Analysis"按钮
2. 跳转到 `/new` 页面
3. 显示分析弹窗，用户上传图片
4. 提交分析，成功后跳转到结果页面
5. 用户可以从结果页面返回Dashboard

### 扩展跳转流程
1. 浏览器扩展捕获图表截图
2. 跳转到Web应用（带扩展标记）
3. 自动检测扩展跳转
4. 自动跳转到 `/new` 页面
5. 分析弹窗自动预填充图片数据

### 认证流程
1. 未登录用户访问 `/new` 页面
2. 自动重定向到登录页
3. 登录后返回 `/new` 页面
4. 继续分析流程

## 部署说明

### 构建和部署
1. 构建命令不变：`npm run build`
2. 部署配置不需要修改
3. 所有路由由前端处理

### 服务器配置
对于生产环境，需要配置服务器将所有路由重定向到`index.html`：
```nginx
location / {
  try_files $uri $uri/ /index.html;
}
```
