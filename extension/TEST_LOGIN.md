# Candlebot 扩展登录测试指南

## 问题描述
之前登录页面存在无限加载问题，Google/GitHub OAuth登录无法使用。

## 解决方案
我们创建了一个简化的登录页面，专注于解决基本登录问题：

1. **修复了无限加载问题**：简化了AuthContext中的认证数据加载逻辑
2. **添加了模拟API支持**：可以在没有后端的情况下测试登录功能
3. **创建了简化的登录页面**：专注于邮箱/密码登录，移除了复杂的OAuth逻辑

## 测试步骤

### 1. 加载扩展到Chrome
1. 打开Chrome浏览器，进入 `chrome://extensions/`
2. 开启"开发者模式"
3. 点击"加载已解压的扩展程序"
4. 选择 `dist` 文件夹

### 2. 测试登录功能

#### 方法一：使用模拟API（推荐）
1. 打开新标签页，会自动显示Candlebot Dashboard
2. 在登录页面，点击"使用模拟API"按钮
3. 使用测试账号登录：
   - 邮箱: `test@example.com`
   - 密码: `password123`
4. 或者点击"测试登录"按钮自动填充并登录

#### 方法二：使用真实API
1. 确保后端API正常运行 (`https://api.candlebot.app`)
2. 在登录页面，点击"使用真实API"按钮
3. 使用您的真实账号登录

### 3. 调试信息
登录页面底部显示调试信息：
- 认证状态（加载中/已登录/未登录）
- 当前用户邮箱
- API模式（模拟API/真实API）

## 技术改进

### 1. AuthContext 修复
- 使用Promise包装chrome.storage API，避免回调地狱
- 移除了复杂的超时和状态管理逻辑
- 添加了模拟API支持，便于测试

### 2. 简化的登录页面
- 移除了OAuth登录按钮（暂时）
- 专注于邮箱/密码登录
- 添加了调试信息和测试功能
- 使用内联样式，避免CSS加载问题

### 3. 模拟API系统
- 提供完整的登录/注册模拟
- 支持用户信息获取
- 可以通过URL参数 (`?mock=true`) 或按钮切换

## 下一步计划

1. **测试基本登录功能**：确保邮箱/密码登录正常工作
2. **逐步恢复OAuth**：在基本登录稳定后，逐步添加Google/GitHub OAuth
3. **优化用户体验**：添加加载状态、错误提示等
4. **集成真实后端**：连接到 `https://api.candlebot.app`

## 故障排除

### 问题：登录页面仍然无限加载
- 检查Chrome控制台是否有错误
- 尝试刷新页面
- 检查扩展是否已正确加载

### 问题：模拟API不工作
- 确保点击了"使用模拟API"按钮
- 检查URL中是否有 `?mock=true` 参数
- 刷新页面后重试

### 问题：真实API连接失败
- 检查网络连接
- 确认 `https://api.candlebot.app` 可访问
- 检查环境变量配置
